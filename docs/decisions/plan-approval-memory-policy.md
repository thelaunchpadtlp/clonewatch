# Decision: Plan Approval Requires Immediate Memory Update

Date: 2026-04-14

## Policy

When the user approves a plan, memory update becomes mandatory as the first execution step.

Required targets:

- `clonewatch.md`
- `docs/project-memory.md`

## Execution order

1. user approves plan
2. memory is updated immediately
3. implementation work continues
4. memory is updated again at closeout if scope shifted

## Why

- preserves continuity between planning and execution
- prevents context drift during long implementation passes
- keeps future sessions decision-complete
